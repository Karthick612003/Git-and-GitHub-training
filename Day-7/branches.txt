🌿 1. Understanding Branches (Parallel Universes)In professional engineering, we never work directly on the main branch. We create a "copy" of the project to test new ideas safely.Create a new branch: git checkout -b <branch-name>Switch between branches: git checkout <branch-name>See all branches: git branch (Local) or git branch -a (All)

☁️ 2. The Remote Workflow (GitHub)You learned that your PC and GitHub are two different "brains." To sync a branch to the cloud:Push the branch: git push origin <branch-name>The Pull Request (PR): On the GitHub website, you "Request" to merge your branch into the main code.The Merge: Clicking the green button on GitHub officially combines the work.The Pull: Your PC doesn't know the website merged until you run:git checkout maingit pull origin main

🧹 3. Cleaning Up (Maintenance)Once a branch is merged, the "scaffolding" is no longer needed.Safe Delete: git branch -d <branch-name> (Only works if merged).Force Delete: git branch -D <branch-name> (Deletes even if unmerged).Cloud Delete: git push origin --delete <branch-name> (Removes from GitHub).
