🟢 DAY 2: The GitHub Bridge
Goal: Connect your local "Physical Vault" (PC) to the "Digital Vault" (Cloud).

1. The Setup (Only do this ONCE per project)
Open Git Bash: Right-click your git-test folder and select "Git Bash Here".

Connect to Cloud: Tell your PC where the GitHub repository is.

Bash
git remote add origin https://github.com/Karthick612003/300-days-automation.git
Note: Always ensure the link ends in .git.

Verify the Bridge: Check if the connection is active.

Bash
git remote -v
2. The Daily "Save & Sync" Flow (Do this EVERY DAY)
To keep your progress consistent, follow this 3-Step Sequence:

Step A: Pack the Box

Bash
git add .
What it does: Collects all new files and changes in your folder.

Step B: Label the Box

Bash
git commit -m "Day 2: Created GitHub repository and linked local folder"
What it does: Saves a permanent "snapshot" with a descriptive note.

Step C: Ship the Box

Pull the README from GitHub to your PC:

Bash
git pull origin main --rebase

Note: The --rebase part tells Git to "put my Day 2 notes on top of the README you just downloaded."

For the VERY first time:

Bash
git push -u origin main
For every day after today:

Bash
git push

💡 Pro-Tips for Consistency:
The "q" Key: If git log shows a : and stops, just press q to quit and go back to typing.

Green Squares: Every time you push, GitHub marks your profile with a green square. Don't break the chain!

Folders: Use mkdir Day-03 tomorrow to keep your different automation topics organized inside the same repository.


If you just realized you have a typo or forgot one sentence in your notes, you can "attach" your new changes to the previous commit so it looks like one perfect save.

Edit and Save your day-2-notes.txt in Notepad.

Stage it: git add .

Amend it:

Bash
git commit --amend --no-edit
--amend: This takes your new changes and "stuffs" them into your last commit.

--no-edit: This tells Git to keep the same message ("Day 2: Created GitHub repository...") so you don't have to re-type it.
