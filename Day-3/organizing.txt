🛠️ DAY 3: Organizing, Cleaning, & Protecting
Today, I moved from a "messy desk" to a "structured system." I learned how to manipulate files effectively, maintain a clean repository, and use professional organization techniques to keep my 300-day journey on track.

📂 The Foundation: Daily Folder Structure
To keep my 300-day journey organized, I am now grouping my work by day.

Create Folder: mkdir Day-03

Create File: touch Day-03/organizing.txt

Add to Git: git add . (Tells Git to track these new items)

🛡️ The .gitignore: The Shield
I implemented a .gitignore file to act as a filter. This keeps junk out of my GitHub repository automatically.

Filename: .gitignore (The dot is required).

Purpose: Instructs Git to ignore temporary/system files (e.g., *.tmp, Thumbs.db).

Benefit: Once a file type is added to .gitignore, Git never prompts me to add those files again.

🧹 Cleaning & Organizing
I learned how to manage my workspace without leaving a trail of "digital dust."

Deleting Files: rm filename.txt

Deleting Folders: rm -r foldername

(The -r flag is vital; it means "recursive," removing the folder and everything inside it.)

Renaming/Moving Files:

For new/uncommitted files: mv old.txt new.txt then git add .

For existing/committed files: git mv old.txt new.txt

Pro-Tip: Using git mv preserves the file’s history!

🏁 The "Day 3" Workflow
Before closing my session, I secured my progress with this final sequence:

Check Status: git status (Verify renames and deletions are correct).

Stage All: git add .

Final Commit: git commit -m "Day 3: Completed organizing, cleaning, and renaming logs"

Sync to Cloud: git push
