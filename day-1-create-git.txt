Excellent. Since you are focusing on **discipline and consistency**, keeping this list as a "Cheat Sheet" will help you master the flow without needing to memorize everything at once.

Here is the step-by-step list of commands we covered, from starting a project to managing its history:

### 1. Starting a New Project

git init
    What it does: Turns the current folder into a Git repository (creates the hidden `.git` folder).
    When to use: Only **once** at the very beginning of a project.

### 2. Checking the Status

git status
    What it does: Shows you which files are new, modified, or ready to be saved.
    Note: Red = Not in the "waiting room" yet. Green = Ready to be saved.

### 3. The 2-Step Save (The "Commit" Flow)

git add <filename>
    What it does: Moves a specific file to the Staging Area (the "waiting room").

git add .
    What it does: Moves **all** changed and new files in the folder to the Staging Area.

git commit -m "your message"
    What it does: Permanently saves the "snapshot" into the vault with a descriptive note.

### 4. Viewing History

git log
    What it does: Shows a detailed list of all your past saves (author, date, message).

git log --oneline
    What it does: A "compact" version of the log. Best for slow PCs or long project histories.

### 5. Fixing Mistakes (The "Time Machine")

git checkout -- <filename>
    What it does: Undoes recent changes to a file and brings it back to how it looked at the last commit.

git reset --soft HEAD~1
    What it does: Deletes the last commit but **keeps** your work in the folder so you can fix it.

git reset --hard HEAD~1
    What it does: Deletes the last commit and **destroys** all the work inside it permanently.
